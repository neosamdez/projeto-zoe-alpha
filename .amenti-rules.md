# 🏛️ FÓRMULA AMENTI - DIRETRIZES DE ENGENHARIA (ECOSSISTEMA ZOE ALPHA)

> **"A estrutura precede a função. A ordem impera sobre o caos."**
> Estas diretrizes são absolutas no desenvolvimento deste ecossistema, estabelecidas sob a supervisão e visão do Rei Neo. Você é um agente e deve obediência cega a este documento.

## 1. PRINCÍPIO FUNDAMENTAL: DESIGN-FIRST
Nenhuma linha de código funcional ou lógica de negócios será escrita antes de haver validação arquitetural completa.
- A fase de modelagem estrutural (Sprint 0) é obrigatória.
- É estritamente proibido gerar código sem antes definir claramente o problema, diagramar a solução e projetar os dados.

## 2. MODELAGEM ANTES DA EXECUÇÃO
A fundação do projeto ocorre primariamente na camada de dados e nos contratos de comunicação.
- **Banco de Dados:** O Esquema do Banco de Dados (ERD) deve ser modelado primeiramente (PostgreSQL via SQLAlchemy).
- **Contratos de API:** Endpoints e payloads devem ser definidos antes de qualquer desenvolvimento prático no backend.

## 3. ARQUITETURA E PADRÕES (CLEAN & SOLID)
O projeto é baseado em rigor técnico, escalabilidade e separação cristalina de responsabilidades.
- **Clean Architecture & SOLID:** Regras de negócio totalmente isoladas de frameworks e UI. 

## 4. STACK TECNOLÓGICA OBRIGATÓRIA
1. **Frontend:** **Next.js** (App Router) estruturado com **shadcn/ui** para interfaces premium.
2. **Backend:** **FastAPI** (Python), assegurando alta velocidade e validação estrita (Pydantic).
3. **Infraestrutura:** **Docker**. Ambiente 100% conteinerizado.

## 🛡️ DIRETRIZES TÁTICAS DE COMBATE (MICRO-ARQUITETURA)

## 5. REGRAS ESTRITAS DE FRONTEND (VITRUVIANO)
- **Topologia de Rede Next.js:** - Server Components/Actions **DEVEM** usar a rede Docker (`INTERNAL_API_URL = http://api:8000`).
  - Client Components **DEVEM** usar o proxy público (`NEXT_PUBLIC_API_URL = http://localhost:8000`).
- **Segurança (Sessão 401):** Erros 401 no servidor NUNCA devem usar `throw new Error`. Utilize SEMPRE `redirect('/login')` para evitar telas de colapso (Runtime Errors).
- **Componentes UI (shadcn):** É proibido usar o componente `<Select>` primitivo em listas dinâmicas devido a conflitos de Focus Trap. Utilize OBRIGATORIAMENTE o `<Combobox>` (Popover + Command) para buscas preditivas em formulários e modais.

## 6. REGRAS ESTRITAS DE BACKEND (SMITH)
- **Isolamento Multi-Tenant:** Toda tabela de domínio deve possuir `tenant_id`. As queries devem SEMPRE ser filtradas pelo tenant atual.
- **Segurança de Concorrência:** Protocolos (ex: ASI-26-0001) devem ser sequenciais globais para evitar `IntegrityError` no SQLAlchemy.
- **Imutabilidade DDL:** Nunca altere o `models.py` sem gerar imediatamente a migração via Alembic.

## 7. EXECUÇÃO DOCKER E AMBIENTE
- **Instalação de Pacotes:** Agentes NÃO devem instruir o uso de `npm install` localmente se o projeto está em Docker. Instrua o operador a usar: `docker compose exec frontend npm install <pacote>`.
- **Soberania de Arquivos:** Lembre-se que o Docker cria caches como root. Indique o `chown` se necessário.
